## Must-do before production

### 1. Build & quality gates
Run locally (or in CI):

```bash
npm run typecheck
npm run lint
npm run build
npm run security:audit
```

`build` is the real gate ‚Äî it catches env/schema issues that `typecheck` alone won‚Äôt.

### 2. Vercel environment variables
Set these in **Production** (mirror from `.env.example`):

| Required | Purpose |
|----------|---------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase client |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | Supabase client |
| `SUPABASE_SECRET_KEY` | Admin/server (never expose to client) |
| `SITE_URL` | Canonical URL ‚Äî **must be your real domain**, not `localhost` |
| `ADMIN_EMAIL` | Single admin gate |

**Strongly recommended for production:**

| Variable | Why |
|----------|-----|
| At least one AI key (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, etc.) | Assistant + copilot |
| `UPSTASH_REDIS_REST_URL` + `UPSTASH_REDIS_REST_TOKEN` | Rate limiting on Vercel (in-memory fallback doesn‚Äôt work across instances) |
| `RESEND_API_KEY` + `EMAIL_FROM_ADDRESS` | Contact form (if you use it) |
| `SENTRY_DSN` + `NEXT_PUBLIC_SENTRY_DSN` | Error tracking |
| `NEXT_PUBLIC_POSTHOG_KEY` | Analytics |
